### Name Rules
- Folder: [**GameVersion**]
- Subfolder/files: [**BlueprintName**]-[**BlueprintFunction**]-[**BlueprintVersion**]-[**BulueprintAuthor**]
> - **Required**: *BlueprintName* 
> - **Optional**: *BlueprintFunction* *BlueprintVersion* *BulueprintAuthor* 
> - It's allowed that single or multiple blueprints exist inside a folder. 
> - If a blueprint works fine, don't overwrite it with new verisons of it. What you should do while updating is to rename the new version with a new *BlueprintVersion*.
> - It's approved to write *README.md* inside a subfolder to introduce a blueprint.
> - It's approved to put pictures inside any subfolder */.picture/* to show details and demos of blueprints.


---

### **Attention**
- The blueprints are cross-platform compatible while the game versions differ by only a small amount.
- The blueprints of version 1.5.x.x are usually available on version 1.6.x.x, but ones of v1.6.x.x can't be loaded by v1.5.x.x usually.
- PC blueprints are usually set as a folder with file *Blueprint.txt* and *Version.txt*, but Android blueprints are usually set as a single file with its showed name. Thus you shold rename the files while using PC blueprints on Android.
