### Naming Rules
- Folder: **[GameVersion]**
- Subfolder/Filenames: **[BlueprintName]-[BlueprintFunction]-[BlueprintVersion]-[BlueprintAuthor]**
> - **Required**: *BlueprintName*  
> - **Optional**: *BlueprintFunction*, *BlueprintVersion*, *BlueprintAuthor*  
> - Multiple blueprints can exist in one folder.  
> - When updating a blueprint, do **not** overwrite the old version. Instead, create a new version with an updated *BlueprintVersion*.  
> - You may add a *README.md* inside a subfolder to describe the blueprint.  
> - Pictures can be placed in `/.picture/` subfolders to show details and demos.

---

### Attention
- Blueprints are generally cross-platform compatible if the game versions are close.  
- Blueprints for version **1.5.x.x** usually work on **1.6.x.x**, but **1.6.x.x** blueprints usually cannot be loaded in **1.5.x.x**.  
- PC blueprints are usually a folder containing `Blueprint.txt` and `Version.txt`. Android blueprints are usually a single file. Rename files accordingly when transferring between platforms.
